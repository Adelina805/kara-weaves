Project Context:

I am designing the frontend UX/UI for the Kara Weaves Textile Design Platform, a web application being developed as part of a summer internship at SDSU ZIP Launchpad.

The platform is intended for professional textile buyers (interior designers, hospitality companies, textile sourcing professionals) who work with Kara Weaves to create custom handwoven textiles.

Current workflow:
Clients communicate customizations through long email chains with Kara Weaves. They discuss stripe layouts, colors, weave types, warp/weft behavior, fabric thickness, and production constraints. Kara Weaves then manually creates mockups and specification sheets before production.

Project goal:
Reduce back-and-forth communication by allowing clients to:
- Design stripe-based woven textiles.
- Preview realistic woven mockups.
- Understand production constraints earlier.
- Generate designs that can later become production specification sheets.

Current MVP priorities:
- Interactive textile editor.
- Accurate woven mockup visualization.
- Warp and weft color editing.
- Stripe-level customization.
- Fabric-specific rendering behavior.
- Simple, intuitive UX.

Technology:
- Next.js
- React
- TypeScript
- Hosted on Vercel
- GitHub for version control

Design philosophy:
- Minimal, editorial, premium aesthetic.
- Inspired by textile sample books rather than SaaS dashboards.
- Calm, spacious interface.
- The textile preview is the hero of the application.
- Professional users value design freedom more than guided presets.

Important:
When suggesting UI changes, prioritize simplicity and usability. Avoid adding unnecessary controls or complexity. The software should feel like a professional textile design tool, not a generic dashboard.
